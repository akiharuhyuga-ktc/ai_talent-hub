# 評価基準

> 出典：CareerLadder_202602.xlsx（2026年2月版）

---

## ラダーの狙い

| 対象 | 目的 |
|------|------|
| **会社にとって** | 「明確なルール × 努力と成長が報われる仕組み」を実現するにあたって、社員へ成長の地図として示すもの |
| **メンバーにとって** | 明確な基準があることで、自身のキャリアを描きやすくなり中長期で活躍・成長（社内価値・市場価値の向上）を目指せるもの |
| **上長にとって** | 公正な評価運用と組み合わせることによって部下の成長支援、ひいては組織力の強化を推進しやすくなるもの |

---

## 評価の基本スタンス

- **総合判定方式**：全項目を満たす必要はなく、業務遂行上必要な項目を上長と確認し、総合的なインパクト（成果）とスキル（職能）で等級を判断する
- **再現性の確認**：たまたま一度できた（ビギナーズラック）ではなく、異なる状況下でも同様のパフォーマンスを発揮できる「再現性」を重視する
- **職能評価と業績評価の分離**：本ラダーは蓄積された「発揮能力（職能）」を評価する指標。一定期間のプロジェクト成否（短期成果）とは切り離して考える（短期成果評価は賞与に影響）

---

## ベース等級定義

等級はメイン等級（1〜6）とサブ等級（1〜5）で構成される。評価は **Impact（仕事の大きさ）** と **Job/Skill（専門性）** の2軸で行う。

### Impact の3軸

| 軸 | 説明 |
|----|------|
| **複雑性** | 課題の構造的な難しさ（手順通り → 統合・再設計） |
| **不確実性** | 情報不足・曖昧さへの対処力（指示に従う → 大局判断） |
| **作用範囲** | 影響が及ぶスコープ（タスク単位 → 全社・体系） |

---

### メイン等級別 定義

#### 等級6：全社けん引
**定義**：全社視点で組織をけん引し、経営幹部/技術幹部としての役割を果たす

| Impact軸 | レベル |
|----------|--------|
| 複雑性 | 仕組みの再設計：組織や事業の大きな仕組みから見直し、根本原因を捉えて効果的な打ち手を設計・実行できる |
| 不確実性 | 大局判断：技術や事業環境の大局的な変化を踏まえて、先を読んだ判断や意思決定を自ら行える |
| 作用範囲 | 全社・体系の主導：「部署〜全社」または「業務体系の変化」を継続的に主導し、経営インパクトを生み出している |

---

#### 等級5：変革と専門性の発揮
**定義**：高度な専門性を活かし、事業や組織、ものづくりに変革をもたらす役割を果たす

| Impact軸 | サブ5 |
|----------|-------|
| 複雑性 | 広範課題の統合：複数ドメインが絡む広範な課題を整理し、関係者を広く巻き込み、複数の論点や制約を統合して解決へ導ける |
| 不確実性 | 意思決定とリスク管理：課題の全体像が見えない状況でも、選択肢を絞り込みリスクを明らかにしながら意思決定できる |
| 作用範囲 | 複数領域の主導：「グループ〜部署」または「複数ドメイン横断の課題」を継続的に主導し、有益な変化を生み出し続けている |

---

#### 等級4：リーダーシップ発揮
**定義**：プロジェクトや専門領域のリーダーシップを発揮して周囲を巻き込み、成果を最大化する役割を果たす

| Impact軸 | サブ4 |
|----------|-------|
| 複雑性 | 領域横断の構造化：担当領域に閉じない複雑な課題の構造を捉え、関係者を巻き込みながら有効な解決策へ導ける |
| 不確実性 | 不確実な要素の解消と整理：要件や制約が入り交じる状況でも、不確実な要素を解消・整理しながら前に進められる |
| 作用範囲 | 担当領域外に波及：「チーム〜グループ」または「隣接領域へ波及する課題」に継続的に働きかけ、成果の最大化に寄与している |

---

#### 等級3：自律的専門家
**定義**：自身の専門性を活かし、担当領域の課題を自律的に解決する役割を果たす

| Impact軸 | サブ4 |
|----------|-------|
| 複雑性 | 構造化と自律遂行：担当領域の課題を構造化し、制約や依存関係を踏まえて進め方を整理しながら自律的に物事を進められる |
| 不確実性 | 探索・仮説立て：情報の不足や矛盾がある状況でも、自ら情報を探索し、妥当な仮説を置いて判断しながら進められる |
| 作用範囲 | 担当領域の安定性：「チーム内」または「担当領域の課題」に主体的に取り組み、安定して課題解決・改善できている |

---

#### 等級2：主体的工夫
**定義**：上位者の支援を前提に、主体的に工夫して業務を遂行する役割を果たす

| Impact軸 | サブ4 |
|----------|-------|
| 複雑性 | 作業の段取り：上位者とコミュニケーションを取りながら、段取りを組んで複雑な作業も遂行できる |
| 不確実性 | 想定外への対処：想定外が起きても、上位者の意図を確認しつつ、自分なりに工夫して対応できる |
| 作用範囲 | 限定的な再現性：「チーム内の限られた範囲」または「基本的な業務領域」で、求められる品質を安定して再現できている |

---

#### 等級1：指示に従い遂行
**定義**：上位者の指導を前提に、指示された業務を遂行する役割を果たす

| Impact軸 | サブ4 |
|----------|-------|
| 複雑性 | 手順通りの作業：やるべきことが明確であれば、その手順に沿って正確に作業できる |
| 不確実性 | 確実な業務の対応：不確実性の低い業務を、指示に沿ってスムーズに進められる |
| 作用範囲 | タスク単位の遂行：「担当タスク単位」または「明確な業務」で、基本的な作業を安定してこなしている |

## 職域別スキル定義

### 職域①：エンジニアリング（ゼネラルスキル）

> エンジニアの包括的な職務遂行能力（技術・開発プロセス・品質・運用）を定義したベース。
> モバイルアプリ開発部では **プロダクトエンジニア** が該当メインとなる。

| 等級 | 期待されるスキル・行動 |
|------|----------------------|
| **6** | 経営・事業戦略と連動した中長期の技術戦略を構想。技術文化の醸成、全社ガバナンス体制を設計・構築。経営層と連携した全体最適な技術投資をリード |
| **5** | 技術戦略の策定と実行。3〜5年先を見据えたアーキテクチャ投資・技術負債返済計画の立案。複数プロダクトの技術方針の整合性を担保 |
| **4** | プロジェクト・専門領域のリーダーシップ発揮。担当領域に閉じない複雑な課題を構造化し関係者を巻き込んで解決。不確実な要素を整理しながら前進できる |
| **3** | 担当領域の課題を構造化し自律的に推進。情報不足の状況でも仮説を置いて判断できる。チーム内での安定した課題解決・改善 |
| **2** | 上位者と連携しながら段取りを組んで遂行。想定外にも自分なりに工夫して対応できる。基本的な品質を安定して再現できる |
| **1** | 指示に沿って正確に作業を遂行。不確実性の低い業務をスムーズに進められる。基本的な作業を安定してこなす |

**プロダクトエンジニアとして特に重視するスキル領域：**
- 技術選定と実装品質（Flutter / KMP / ネイティブの適切な選択）
- 開発プロセスへの貢献（PoC実装、コードレビュー、テスト）
- プロダクト品質と安定稼働への責任
- 共通資産・知見の横断活用・共有

---

### 職域②：PM / BizDev（ゼネラルスキル）

> モバイルアプリ開発部では **Producer（企画・プロデューサー）** が該当。

| 等級 | 期待されるスキル・行動 |
|------|----------------------|
| **6** | 全社ポートフォリオを俯瞰した事業構想・中長期戦略の策定。経営層や外部幹部との対等な交渉。新たな事業機会の探索・創出 |
| **5** | 持続可能で競争力あるプロダクト戦略を自ら描き事業目標達成を牽引。ROI最大化のKPI管理。会社間・部門間の利害調整と全体最適な合意形成 |
| **4** | プロダクトビジョンに基づく開発ロードマップ策定と関係者合意形成。リスク・スコープを自律的に制御してデリバリー完遂。潜在リスクの事前特定・評価 |
| **3** | 担当領域の課題を構造的に整理し自律的に前進させる。技術的制約を理解した上での要件定義。適切な会議体運営と透明性の高いレポーティング |
| **2** | 上位者の意図を理解した正確なデータ集計・市場分析。指示された要件の仕様化・ドキュメント化。KPIのモニタリングと異常値の報告 |
| **1** | 指示された範囲での市場調査・ドキュメント作成補助。基本的な動作確認や資料整理を期限内に遂行。適切な報連相と学習継続 |

**来期のProducerとして特に重視するスキル領域：**
- 仮説起点の企画提案力（顧客課題の発見・構造化）
- PoC要件の定義と検証設計
- 部長会デモに向けたストーリー構築・プレゼンテーション
- ステークホルダーとの信頼関係構築

## 運用ガイドライン

### 評価の進め方
1. **職域の確認**：エンジニア or Producer（PM/BizDev）のどちらが主職域かを確認する
2. **等級の確認**：現在のメイン等級・サブ等級を確認する
3. **Impact軸の評価**：複雑性・不確実性・作用範囲の3軸で現状を把握する
4. **Job/Skillの評価**：職域別スキル定義と照らし合わせて専門性を評価する
5. **総合判定**：インパクトとスキルを総合的に判断して等級を決定する

### 特殊ケースの対応
- **複数職域をまたがるケース**：メイン職域を定めた上で、任意の職域ラダーをサブとして参照できる
- **職域に当てはまらないケース**：「等級定義」と「Impact（仕事の大きさ）」を参照する
- **キャリアチェンジ**：当人と上長・人事が事前に相談確認した上で、新しい職務にあわせて参照する職域を変更できる
